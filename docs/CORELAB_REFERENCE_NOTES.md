# CORE LAB 사업비 기능 참고 메모

분석 대상: https://corelab-2026.web.app/login

2026-06-26에 로그인 후 `내 연구회 > 사업비`, `예산 사용법`, `정산보고서 다운로드` 흐름을 확인했다.

## 사이트 구조

- React 단일 페이지 앱
- Firebase Authentication
- Firestore 기반 데이터 저장
- 연구회별 상세 페이지 안에 탭형 기능 배치
- 정산보고서 다운로드는 Excel `.xlsx`

## 사업비 탭 구성

상단 버튼:

- 예산 편집
- 집행 계획 등록
- 집행 결과 입력
- 정산보고서 다운로드

예산표 컬럼:

- 비목
- 최초예산
- 현재예산
- 사용액
- 잔액
- 사용률

CORE LAB 비목:

- 일반수용비
- 공공요금및제세
- 특근매식비
- 임차료
- 기타운영비
- 국내여비
- 사업추진비

우리 앱은 아래 3개로 축소한다.

- 연구활동비
- 직접성 경비
- 업무협의회비

## 집행 계획 등록 UX

입력 필드:

- 비목
- 상태: 계획 / 집행 완료
- 사용 내용
- 계획 등록일
- 계획 금액

사이트 안내 문구:

> 한 지출 항목은 ‘집행 계획’으로 먼저 등록하고, 실제 집행되면 아래 전체 지출 내역에서 그 줄의 ‘집행 결과’ 버튼으로 전환하세요. 계획과 결과를 따로 등록하면 사용액이 중복 합산되니 주의하세요.

우리 앱 적용:

- “계획” 기능은 1차 버전에서는 선택 기능으로 둔다.
- 실제 정산은 영수증 제출 중심이므로 기본은 “집행 완료” 입력으로 시작한다.
- 계획 기능을 넣는다면 계획과 결과를 같은 건으로 연결하는 UI가 필요하다.

## 집행 결과 입력 UX

상태를 `집행(지출 완료)`로 바꾸면 아래 필드가 추가된다.

- 지출 등록일
- 지출 금액
- 품명
- 지급처(상호)
- 단위
- 수량
- 단가
- 결제방법: 카드결제 / 계좌이체
- 증빙번호

우리 앱 적용:

- 이 필드는 그대로 가져오면 좋다.
- 추가로 `제출자`, `첨부파일`, `증빙 누락 상태`, `반려 메모`가 필요하다.

## 예산 사용법 탭

탭 구성:

- 개요
- 비목별 기준
- 국내여비 정액표
- 집행 원칙
- 정산·증빙

우리 앱 적용:

- `도움말` 또는 `정산 기준` 탭을 둔다.
- 항목별 카드에 “사용 가능 / 주의 / 필요 증빙”을 보여준다.
- 직접성 경비, 연구활동비, 업무협의회비 기준을 Padlet 정산서 양식 기준으로 정리한다.

## 공통 집행 원칙 참고

CORE LAB 사이트 원칙:

- 계좌이체·사업비카드 집행
- 협약기간 내 집행 완료
- 사업 목적·용도 내 사용
- 비목 변경·이월은 사전 승인
- 금지 업종·시간대 사용 불인정
- 환급 가능 세액 제외

AI 동아리 앱 적용:

- 현금/간이영수증 경고
- 사업 기간 밖 지출 경고
- 목적 외 물품 경고
- 기자재 등 자산취득성 물품 경고

## 정산·증빙 참고

CORE LAB 사이트는 집행 상세를 입력해두면 정산보고서를 자동 생성한다.

우리 앱에서 필요한 증빙:

- 공통: 카드영수증 또는 계좌이체 확인증
- 직접성 경비: 구매내역서, 해외 결제 시 국내 카드사 원화 전표
- 업무협의회비: 협의록, 참석자, 회의 내용, 사진
- 연구활동비: 확인서, 결과물, 계좌이체 확인증

## 정산보고서 다운로드

CORE LAB 사이트의 정산보고서 다운로드 결과:

- 파일 형식: `.xlsx`
- 포함 문서:
  - 사업비 사용실적 보고서
  - 정산총괄표
  - 사업비 총괄표 및 집행내역
  - 세부비목별 집행내역서

우리 앱 출력 우선순위:

1. 인쇄용 HTML/PDF
2. XLSX
3. PPTX 정산서 양식 자동 채우기

## 우리 앱에 추가하면 좋은 것

- 영수증 이미지/PDF 미리보기
- 첨부 누락 자동 경고
- 제출자별 상태 관리
- 반려 사유 메모
- 항목별 자동 증빙번호
- JSON 내보내기/가져오기
- 최종 출력 전 점검표
