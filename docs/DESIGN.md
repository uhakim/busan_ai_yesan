# AI 디지털 교사 동아리 정산 도우미 설계

## 목적

AI 디지털 교사 동아리 정산 업무에서 선생님별 영수증, 지출 내역, 증빙 파일을 모으고 검토한 뒤 정산서 양식에 맞춰 최종 출력물을 만드는 도구다.

## 기본 방향

- CORE LAB 사이트의 사업비 탭 UX를 참고하되, 우리 동아리 정산은 훨씬 단순하게 만든다.
- 예산 항목은 3개로 제한한다.
  - 연구활동비
  - 직접성 경비
  - 업무협의회비
- 총 예산은 2,000,000원이다.
- 연구활동비 한도는 1,000,000원이다.
- 업무협의회비 한도는 200,000원이다.
- 직접성 경비는 별도 상한보다 정산 가능 증빙과 사업 목적 부합 여부를 우선 확인한다.

## 사용자

### 제출자

동아리 구성원 선생님.

해야 할 일:

- 지출 정보 입력
- 영수증 또는 계좌이체 확인증 첨부
- 해외 AI 구독이면 원화 카드 전표 첨부
- 업무협의회비면 협의록 또는 참석 증빙 첨부

### 관리자

유하님 또는 동아리 총무.

해야 할 일:

- 제출 내역 검토
- 누락 증빙 확인
- 항목별 합계 확인
- 오류 또는 반려 메모 작성
- 최종 정산서 출력

## 화면 구조

### 1. 대시보드

상단 카드:

- 총 예산 2,000,000원
- 사용액
- 잔액
- 사용률
- 누락 증빙 건수

항목별 표:

| 항목 | 한도 | 사용액 | 잔액 | 상태 |
|---|---:|---:|---:|---|
| 연구활동비 | 1,000,000 | 자동 계산 | 자동 계산 | 정상/초과 |
| 직접성 경비 | - | 자동 계산 | - | 증빙 확인 |
| 업무협의회비 | 200,000 | 자동 계산 | 자동 계산 | 정상/초과 |

### 2. 선생님 제출 화면

필수 입력:

- 제출자 이름
- 학교
- 동아리명
- 예산 항목
- 사용 내용
- 지출일자
- 지출 금액
- 결제방법: 카드결제 / 계좌이체 / 기타
- 품명
- 지급처 또는 상호

선택 입력:

- 단위
- 수량
- 단가
- 비고
- 증빙번호 직접 입력

첨부:

- 카드 영수증
- 계좌이체 확인증
- 거래명세서
- 외화 영수증
- 국내 카드사 원화 전표
- 협의록
- 활동 사진

### 3. 관리자 확인 화면

기능:

- 제출자별 필터
- 항목별 필터
- 증빙 누락 필터
- 금액 초과 경고
- 미확인/확인/반려 상태
- 반려 사유 메모
- 증빙 파일 미리보기

### 4. 최종 출력 화면

정산서 양식에 맞춘 인쇄용 화면을 만든다.

출력 섹션:

- 정산서 표지
- 운영비 실적 총괄표
- 연구활동비 지출 세부내역
- 직접성 경비 지출 세부내역
- 업무협의회비 지출 세부내역
- 항목별 증빙자료 묶음

출력 방식:

- 1차: 브라우저 인쇄로 PDF 저장
- 2차: XLSX 자동 생성
- 3차: PPTX 양식 자동 채우기

## 검증 규칙

### 공통

- 금액은 0보다 커야 한다.
- 지출일자는 사업 기간 안이어야 한다.
- 사용 내용과 지급처는 비워둘 수 없다.
- 카드결제는 카드 영수증이 필요하다.
- 계좌이체는 이체확인증이 필요하다.
- 간이영수증은 경고한다.
- 자산취득성 물품은 경고한다.

### 연구활동비

- 총액 1,000,000원 초과 시 경고한다.
- 수당류는 피지급자, 지급 기준, 계좌이체 확인증이 필요하다.

### 직접성 경비

- AI 구독료, 도서, 인쇄비 등 사업 목적 직접 관련 지출만 허용한다.
- 해외 AI 구독이면 외화 영수증과 국내 카드사 원화 전표를 모두 요구한다.
- 구독 이용 기간이 2026년 12월까지인지 확인한다.

### 업무협의회비

- 총액 200,000원 초과 시 경고한다.
- 식비/다과비만 가능하다고 안내한다.
- 협의록, 참석자, 일시, 장소, 회의 내용, 사진 첨부를 요구한다.

## 구현 우선순위

1. 로컬 단일 HTML 프로토타입
2. JSON 저장/불러오기
3. 영수증 이미지/PDF 미리보기
4. 항목별 집계와 경고
5. 인쇄용 정산서 출력

## 보안 메모

- 영수증, 계좌, 카드전표는 개인정보와 금융정보가 포함될 수 있다.
- 초기 버전은 공개 웹 배포하지 않는다.
- 배포가 필요하면 Google 로그인, 업로드 권한, 파일 접근 범위를 먼저 설계한다.
